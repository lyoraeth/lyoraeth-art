## Часть 2. Микрокопия для форм (i18n-строки)

Текущая строка `contact.privacy` отсылает к политике — этого под новые правила мало, нужна явная формулировка согласия с целью и ссылкой на отдельный документ. Готовые значения для подстановки в локали:

### Контактная форма

```json
"contact": {
  "privacy": "Отправляя форму, я даю согласие на обработку указанных данных и их трансграничную передачу для ответа на обращение — см. согласие и политику.",
  "privacy_en": "By submitting, I consent to the processing and cross-border transfer of the data I provide so my enquiry can be answered — see consent and policy."
}
```

### Форма комментариев

```json
"post": {
  "comments": {
    "privacy": "Отправляя комментарий, я даю согласие на обработку никнейма и текста и их публикацию — см. согласие и политику.",
    "privacy_en": "By submitting, I consent to the processing and publication of my nickname and comment — see consent and policy."
  }
}
```

**Реализация (важно под 01.09.2025):**
- Слова «согласие» и «политика» — две отдельные ссылки на два отдельных документа.
- Галочку согласия лучше делать **неотмеченной по умолчанию** (или фиксировать факт отправки как действие-согласие — допустимо, но явная галочка крепче).
- Не объединяйте согласие на ПДн с другими условиями в одном чекбоксе.
